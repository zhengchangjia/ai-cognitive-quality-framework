# AI Cognitive Quality Framework (ACQF)

**一个用于评估AI生成分析报告质量的认知维度框架。**

当AI越来越多地承担分析任务——从行业研究到政策解读——一个紧迫的问题是：**我们如何判断这些分析的质量？**

ACQF 提供了一个系统性的答案。它不依赖感觉或笼统的“好/坏”判断，而是从9个维度、40+个子维度，对分析报告进行精细化的认知质量评估。

### 为什么需要这个框架？

传统的评估标准有一个致命缺陷：**安全但平庸的报告往往能拿高分。** 一份报告可以完整复述素材、逻辑自洽、格式规范，却没有提供任何读者仅读素材无法获得的理解。

ACQF 通过以下设计解决这个问题：

- **认知增量维度**专门识别“复述素材”的伪分析，要求报告提供超越素材的原创洞见
- **否决规则**确保在关键维度上的严重缺陷不会被其他维度的高分掩盖
- **边界偏移度**惩罚那些在不该精确的地方给出虚假精确数字的报告

### 评估维度

| 维度 | 权重 | 核心问题 |
|:---|:---|:---|
| 认知增量 | 20% | 报告是否提供了仅读素材无法获得的理解？ |
| 外部逻辑一致性 | 10% | 推论的因果方向是否正确？是否与素材事实矛盾？ |
| 思维严谨程度 | 10% | 归因是否深入？矛盾是否被识别和处理？ |
| 指令遵循度 | 10% | 是否完整回答了所有问题？格式是否符合要求？ |
| 素材引用度 | 10% | 关键论断是否有素材支撑？量化数据是否准确？ |
| 边界偏移度 | 10% | 是否有越界推断？结论的绝对化程度如何？ |
| 内部逻辑一致性 | 10% | 报告内部是否前后矛盾？综合结论与分点是否一致？ |
| 幻觉程度 | 10% | 是否存在事实或信源捏造？ |
| 选择性偏差 | 10% | 是否忽视了重要反例？证据选取是否偏倚？ |

每个维度包含4-5个子维度，每个子维度有0-10分的6级评分标准。

### 快速开始

1. 阅读 `SPEC.md` 了解完整的评估标准
2. 准备一份AI生成的分析报告和对应的原始素材
3. 将评估标准和分析报告一起发送给你信任的大模型
4. 获取9维度评估结果

详细的提示词模板见下文。

### 提示词模板

复制 `tools/prompt-template.md` 中的完整提示词，将 `{MATERIAL}` 替换为原始素材，`{REPORT}` 替换为待评估的分析报告，发送给你信任的大模型即可。
